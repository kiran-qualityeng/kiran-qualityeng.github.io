Order Workflow — Prevent Invalid Transitions & Cancellation Handling (ST-OW)

Priority: High
Focus: Workflow correctness & audit trail
  1. ST-OW-001 — Invalid Order Status Changes
    1. Description: Verify system prevents invalid order status transitions
    2. Preconditions: Order management UI/API with status enum (PENDING, CONFIRMED, SHIPPED, DELIVERED).
    3. Steps:
      1. As admin, navigate to order management
      2. Find order with status: PENDING
      3. Attempt to change status directly to: SHIPPED
      4. Attempt to change status directly to: DELIVERED
      5. Follow valid flow: PENDING → CONFIRMED → SHIPPED → DELIVERED
    4. Expected Results:
      • Invalid transitions show error: "Cannot ship order that is not confirmed"
      • Valid transitions work successfully
      • Audit trail maintained for all status changes
  2. ST-OW-002 — Cancel Order in Various States
    1. Description: Verify cancellation works correctly in different order states
    2. Preconditions: Cancellation operation exists and inventory service connected.
    3. Steps:
      1. Cancel order in PENDING state
      2. Cancel order in CONFIRMED state
      3. Cancel order in SHIPPED state
      4. Cancel order in DELIVERED state
    4. Expected Results:
      • PENDING/CONFIRMED: Cancel successful, inventory updated
      • SHIPPED: Error "Cannot cancel shipped order"
      • DELIVERED: Error "Cannot cancel delivered order"
